git ignore를 사용하는 이유

 

-파일 백업 정보,제품 빌드 정보 등의 로컬 구성 재정의들을 저장소에 올려놓지 않기 위함

-원하지 않는 파일이 upload되는 것을 방지 할 수 있다.

-혹여나 발생할 수 있는 데이터 충돌로 인한 손실을 방지 할 수 있다.

 

git ignore 하는 파일과 이유

 

-venv: 운영체제 마다 가상환경 폴더의 구성이 다르므로 os독립성을 유지하기 위해서 가상환경 폴더를 제외 시켜야한다.

-settings.py: 프로젝트 폴더 안에 있는 settings.py는 SECRET_KEY 변수가 존재하는데, 이 키를 노출 시 암호화된 로직

이나 데이터에 접근할 가능성을 높여주기 때문에 제외 시켜야 한다. 

-pycharm 설정폴더 .idea: 특별히 프로젝트 설정을 보존하고 싶은게 아니라면 제외시킨다.

-db.sqlite3:실제 데이터 파일이 들어가 있는 데이터 베이스 파일은 필요가 없으므로 제외 시켜야 한다.

-각종 로그 파일 및 임시 파일 또한 필요 없기 때문에 제외한다.
